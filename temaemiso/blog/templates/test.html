注意事項①

本ページでは「temaemiso」というプロジェクト名で作成を進めています。
各コマンドの「temaemiso」という部分は任意のプロジェクト名に置き換えてお読みください。

本ページではPoetry環境上にDjango環境をセットアップする手順となります。
そのため、Django起動などのコマンドが公式ドキュメントとは異なります。
※Poetry上ではPythonスクリプトを起動する方法が異なるためです。

Djangoのインストール

Poetryの依存関係にDjangoを追加し、インストールする。

poetry add django
Postgresを設定する場合は以下のコマンドも実施
poetry add psycopg2

設定しない場合は飛ばしてよいです。
postgresqlの設定

Djangoプロジェクトの開始

以下コマンドでDjangoプロジェクトを作成・開始します。
※プロジェクトが何かについては後述の「Djangoアプリケーション設定」にて触れます。
Django公式ドキュメント(project)でも言及があります 

poetry run python django-admin startproject temaemiso
すでにPoetryやgitの設定などですでにディレクトリがある場合

ディレクトリ内で以下のコマンドを実行してください。

poetry run python django-admin startproject myproject .

カレントディレクトリをプロジェクトのディレクトリとしています。
これにより、余分な階層を作らないようにします。

以下のようなディレクトリが生成されています。
※現時点では詳細な理解は不要（詳細を知りたい方はこちら）

repository_name/
manage.py       # プロジェクトに対する様々な操作を行うユーティリティ
temaemiso/      # プロジェクトのPythonパッケージ
	__init__.py # このディレクトリがPythonパッケージであることを知らせる
	settings.py # プロジェクトの設定ファイル
	urls.py     # プロジェクトのURL宣言（プロジェクトの目次のようなもの）
	asgi.py     # ASGI互換Webサーバのエントリポイント
	wsgi.py     # WSGI互換Webサーバのエントリポイント

Djangoの起動確認

以下コマンドでDjango開発サーバーを起動

poetry run python manage.py runserver
ここでエラーになってしまう場合、前述の手順をご確認ください。

考えられるエラー原因

  • コマンドの打ち間違い
  • 前提条件の間違い※本ページはPoetry x Django 環境の構築手順です。
  • Poetryの設定・起動の誤り※Poetry設定をご確認ください。

Djangoアプリケーション設定

1. アプリケーションの作成

「アプリとかプロジェクトとか一体なんなんだ。。。」という気持ちはいったんおいて、
以下コマンドでblogアプリケーションを作成します。

mkdir temaemiso/blog
poetry run python manage.py startapp blog temaemiso/blog
Caution

今回はサブモジュールとしてblogアプリケーションを作成したかったため
事前にディレクトリを作成しています。

トップレベルモジュールとして設定したい場合

アプリケーションはPythonパス上であればどこにでも配置できます。
トップレベルモジュールとして設定したい場合は以下コマンドを使用してください。

python run python manage.py start app blog
プロジェクトとアプリケーション

プロジェクトとアプリの違いは何でしょうか？ アプリとは、ブログシステム、公的記録のデータベース、小規模な投票アプリなど、何かを行う Web アプリケーションです。プロジェクトは、特定のウェブサイトの構成とアプリのコレクションです。プロジェクトには複数のアプリを含めることができます。 アプリは複数のプロジェクトに存在できます。
※公式ドキュメント（プロジェクトとアプリケーション）より引用

厳密には違うと思いますが、以下のような最初の段階では
以下の認識でいればよいかと思います。

  • Project: サイト全体
  • app: 各種タブに設定されたページや機能

# blog/views.py
from django.http import HttpResponse

return HttpResponse("Hello, world. You're at the blog index.")

3. URLの対応付け

blog配下にurls.pyというファイルを作成し、以下の通りに編集する。

# blog/urls.py
from django.urls import path
from . import views

urlpatterns = [
path("", views.index, name="index")
]

次にプロジェクトのURLにモジュールの記述を反映させます。
temaemiso配下のurls.pyを以下の通りに編集します。 

# temaemiso/urls.py
from django.contrib import admin
from django.urls import include, path

urlpatterns = [
path("blog/", include("temaemiso.blog.urls")), # モジュール階層に注意
path("admin/", admin.site.urls),
]
Caution

includeに読み込ませる際に、モジュール階層を意識しないと、エラーが出てしまう。

Module not found Error.

Djangoの各種設定

ここではDjangoの各種設定を行っていきます。
推奨・任意の設定項目ですが、要否をご確認いただきながら進めてください。

1. HTMLテンプレート設定（超絶推奨）

こんな人は設定してください。
  • モダンなシステムを利用し、HTMLを部品化し、再利用したい
  • 結構たくさんなページを作成予定人
  • 「テンプレートシステムってなんや？」の人

a. ディレクトリの作成

アプリケーション（本ページではblog）のディレクトリ直下に
templatesディレクトリを作成します。 

mkdir temaemiso/blog/templates

b. プロジェクト設定ファイルの編集

上記で作成したディレクトリをteamaemiso/settings.pyのTEMPLATES辞書に追加する。 

TEMPLATES = [
{
	"BACKEND": "django.template.backends.django.DjangoTemplates",
	"DIRS": [
		# blog用templateフォルダ
		BASE_DIR.joinpath("ap_login").joinpath("templates") # ★追加
	],
	"APP_DIRS": True,
	"OPTIONS": {
		"context_processors": [
			"django.template.context_processors.debug",
			"django.template.context_processors.request",
			"django.contrib.auth.context_processors.auth",
			"django.contrib.messages.context_processors.messages",
		],
	},
},
]

c. テンプレートファイルの作成

次に、blog/templates/blog_base.htmlを作成し、以下のコードを記述する。 

<!DOCTYPE html>
<html>
<title>{{ blog_title }}</title>
<head></head>
<body>
	<div>
		<h1>blog_title_here: {{ blog_title }}</h1>
		<div>
			<p>paragraph</p>
		</div>
	</div>
</body>
</html>

d. 画面のレンダリング（テンプレートへの値の反映&レスポンス）

最後にblog/views.pyを以下のように編集してください。

from django.http import HttpResponse
from django.template import loader


def index(request):
template = loader.get_template("blog_base.html")  # テンプレートをロードする
context = {
	"blog_title": "手前みそ",
}  # テンプレートに当てはめる値
# HTTPレスポンスとして返却
return HttpResponse(template.render(context, request))

a. PostgreSQLのインストール

以下の手順に沿って、お手持ちのPCへPostgres

  • PostgreSQLのインストール

b. PostgreSQLにDBを作成

pgAdmin4（PostgreSQLのGUI）を起動するなどして、
PostgreSQL上にDBを作成します。

PosstgreSQL上に、DjangoからアクセスするDBを作成します。

  • windowsキーを押して「pgAdmin」と打つと検索できます。

  • 「Login/Gorup　Roles」を左クリックし、「Create」 > 「Login/Group Role...」をクリック

  • 任意のユーザ名を入力

  • 「Privileges」タブへ移動し、「Can login?」にチェックを付けて、「Save」する

  • 「Dadtabase」を左クリックし、「Create」 > 「Database...」をクリック

-　Ownerに先ほど作成したユーザ名を設定

3. 管理者設定

a. migrationを実行

以下のコマンドを実行します。

poetry run python manage.py migrate

migrate コマンドは INSTALLED_APPSの設定を参照するとともに、 temaemiso/settings.py ファイルのデータベース設定に従って必要なすべてのデータベースのテーブルを作成します。

3. 管理者ユーザの作成

a. 管理者ユーザの作成

以下のコマンドでプロジェクトに管理者ユーザを作成します。

poetry run python manage.py createsuperuser

管理者の名前・メールアドレスは任意のユーザー名を入力します。

Username: tanukichi
Email address: tanukichi@example.com

最後はパスワードの入力です。確認のため2回同じ入力を求められます。

Password: ************
Password (again): ***********
Superuser created successfully.

「Superuser created successfully.」というメッセージが表示されていればOKです

b. 管理画面へのログイン

Djangoでは特に設定していなければデフォルトで管理画面が利用できます。
管理画面ではDBへのデータの登録や認証したユーザの管理などを行うことができます。

以下のコマンドでDjangoを起動してください。

poetry run python manage.py runserver

起動後、管理画面へアクセスしてください。

上記のような画面が出てきますので、
「a. 管理者ユーザの作成」で作成したユーザ情報でログインしてください。
